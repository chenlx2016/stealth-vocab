## 📄 潜行者词库 (Stealth Vocab) MVP 需求文档 V1.0

### I. 🎯 产品目标与愿景

| **属性**           | **描述**                                                                |
| ------------------------ | ----------------------------------------------------------------------------- |
| **产品愿景**       | 在不打扰工作的前提下，将办公室的微小间隙转化为高效、隐蔽的雅思词汇学习时间。  |
| **产品形态**       | 浏览器侧边栏/浮窗微组件（Browser Sidebar/Overlay Widget）。                   |
| **MVP 核心目标**   | 验证用户在工作流中**主动且安全**地使用碎片时间进行复习的行为模式。      |
| **成功指标 (MVP)** | 核心用户（每周使用 5 天以上）每日平均主动唤醒微组件的次数**$\ge 10$**次。 |

### II. 👥 目标用户与核心痛点

**目标用户：** 准备雅思考试的上班族或案头工作者。

**用户核心痛点：**

1. 缺乏整块学习时间，传统 App  **切换成本高** 。
2. 在工作时使用学习工具，有 **被发现的焦虑感** 。
3. 需要**高效利用** 10-30 秒的短暂等待或切换时间。

### III. ⚙️ MVP 核心功能与流程（聚焦 Flow 2 & 3）

| **编号** | **功能模块**       | **功能点描述（MVP ONLY）**                                                                | **关键交互与实现要求**                                                                                                                                                 |
| -------------- | ------------------------ | ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **C1**   | **核心唤醒/隐藏**  | **全局快捷键呼出与收起。**                                                                | **默认快捷键：**$\text{Ctrl} + \text{Shift} + \mathbf{V}$**(Vocab)。**必须确保呼出和收起为 **瞬间动作** （加载时间**$\le 100\text{ms}$**）。 |
| **C2**   | **瞬时上下文恢复** | 界面收起后，再次唤醒时，**精确恢复到中断前的单词和状态** 。                               | 必须实时存储：当前单词 ID、是否已翻面（显示英文）、当前复习列表索引。                                                                                                        |
| **C3**   | **极简复习界面**   | 窄高的侧边栏浮窗，低对比度深色 UI。                                                             | 仅包含中文释义、英文显示区、三态反馈按钮和极简进度条。**严禁使用任何花哨的动画或音效。**                                                                               |
| **C4**   | **自测与翻面**     | 首次只显示**中文释义** 。用户点击或按**空格键**后，显示 **英文单词和音标** 。 | 交互必须流畅，无加载延迟。                                                                                                                                                   |
| **C5**   | **三态反馈与跳转** | 提供 `✅ 掌握`、`❓ 模糊`、`❌ 遗忘`三种状态按钮。                                        | 用户点击反馈按钮后：**1. 状态实时存储；2. 进度更新；3. 立即跳转至下一个词。**                                                                                          |
| **C6**   | **简易复习算法**   | 决定下一个展示的单词。                                                                          | **MVP 简易规则：**优先展示用户上次标记为**“遗忘”**的词汇。如果没有“遗忘”词，则按固定顺序展示。                                                                     |
| **D1**   | **进度与数据**     | 仅显示“今日已复习 X 个词”的极简数字。                                                         | 数据必须实时持久化（本地或云端），确保关闭浏览器后进度不丢失。                                                                                                               |

### IV. 🛠️ MVP 用户使用流程 (User Flow)

#### 流程 1: 【侧边栏 - 碎片复习】 (工作流微组件)

1. **[时机]** 用户完成一个工作任务，或等待程序加载（ **S2 切换间隙** ）。
2. **[启动]** 用户按下  **$\text{Ctrl} + \text{Shift} + \text{V}$** 。
3. **[状态]** 侧边栏瞬间滑出，显示系统判定为“最需复习”的词汇 **中文释义** （C6）。
4. **[自测]** 用户默想英文，按**空格键**翻面（C4）。
5. **[反馈]** 用户评估掌握度，点击  **三态按钮** （C5）。
6. **[循环]** 界面即刻自动加载下一个需复习的词汇，重复步骤 4-5。
7. **[退出]** 用户需要立即返回工作，再次按下 **$\text{Ctrl} + \text{Shift} + \text{V}$** 或点击 **退出按钮** 。
8. **[收起]** 侧边栏瞬间收回， **当前状态被保存** （C2）。

#### 流程 2: 【瞬时隐藏与安全保障】

1. **[中断]** 用户正在流程 1 中进行复习。
2. **[触发]** 用户按下  **$\text{Ctrl} + \text{Shift} + \text{V}$** 。
3. **[结果]** 侧边栏 **瞬间收回** ，只在背后留下 **持久化的数据** （C2, D1）。
4. **[恢复]** 用户再次按下  **$\text{Ctrl} + \text{Shift} + \text{V}$** 。
5. **[结果]** 侧边栏瞬间弹出， **显示中断前的那个词汇** （C2）。
---
### V. ⚠️ 非功能性需求与风险

1. **性能要求：** 启动、隐藏、翻面、跳转的延迟必须小于  **150 毫秒** 。
2. **兼容性：** 必须兼容主流 Chromium 内核浏览器（Chrome, Edge）。
3. **快捷键冲突处理：** 在 MVP 验证中，需**重点测试** **$\text{Ctrl} + \text{Shift} + \text{V}$** 与主流办公软件（如：Office Web 版、Slack Web 版、Google Docs）的冲突情况。
4. **数据安全：** 必须确保用户状态数据（C2, D1）存储的 **可靠性** ，以避免挫败感。
